Projecting Fantasy Football Points Using Current Game Status¶

Alec Rowell¶

December 2022¶

1. Introduction¶

1.1 What is Fantasy Football?¶

Fantasy football is an online game for people who watch the NFL in which competitors form leagues (typically of size 8-16) with friends and draft NFL players to compete for them. Each NFL play awards points to fantasy players based on their performance on a given play. There are a few different popular scoring methods, but in this project, I will focus on scoring from a typical "PPR" league, which is as follows:

  • 1 point to the receiver for a reception
  • 0.1 points per receiving yard
  • 0.1 points per rushing yard
  • 6 points for a rushing or receiving touchdown
  • 0.04 points per passing yard
  • 4 points for a passing touchdown

There are a few other ways to gain or lose points (such as fumbling) but for the purposes of this project, I'll stick to the list above. Also, we are only going to be dealing with receiving, rushing and passing, so defenses and kickers will be excluded.

1.2 Projections¶

To help players make roster decisions, fantasy websites provide a projected point value for each player. As the games are in progress, players can view an updated projection for each player as well as for their whole team in order to see how they are stacking up to their competition. This in-game projection is calculated by dividing the projection by the portion of time left in the game. However, this system is clearly flawed and could be improved upon. Creating a more accurate live projection is the intent of the project.

2. Acquiring Data¶

2.1 nfl-data-py¶

For the majority of the data in this project, I will be using a Python package called nfl-data-py. Information about this dataset can be found at https://pypi.org/project/nfl-data-py/. \ This package is a way to access data from multiple previously collected datasets as outlined on the page above. It will make it very easy for us to access and manipulate most of our data. To begin, I will be importing their table for general team descriptions for visualizations, the table for rosters, and as the play-by-play data from 2012-2021 which I will use for the majority of our analysis.

2.2 FantasyPros Projections¶

FantasyPros is a popular site that is well regarded among serious fantasy players. They publish weekly projections for each player that we will use to test our models. However, only the projections from the current season are available to scrape, so we will only have 13 weeks worth of data to test on. On the other hand, this provides an easy way for us to divide our training data and test data since we will only test on the 2022 season. \ We will use the requests package and pandas' readhtml function in order to parse this data.

url = 'http://www.fantasypros.com/nfl/projections/qb'
qb_proj = pd.DataFrame()
for week in range(1, 14): # Iterate over weeks 1-13
  params = {'week': week, 'scoring': 'PPR'}
  resp = requests.get(url, params=params)
  df = pd.io.html.read_html(resp.text, attrs={'id': 'data'})[0] # Parse the html
  df['week'] = week # Add column for week
  qb_proj = pd.concat([qb_proj, df]) # Concat to our dataframe
